# Authentication Module Documentation

## Overview

This module handles all authentication flows using **JWT (JSON Web Tokens)** via the `tymon/jwt-auth` package. It follows a modular architecture with Repository, Service, and Controller layers, using interfaces for dependency injection.

---

## Installation

```bash
composer require tymon/jwt-auth
php artisan vendor:publish --provider="Tymon\JWTAuth\Providers\LaravelServiceProvider"
php artisan jwt:secret
```

---

## Environment Variables

Add the following to your `.env` file:

```env
JWT_SECRET=<auto-generated by php artisan jwt:secret>
JWT_BLACKLIST_ENABLED=true
```

---

## Configuration

### `config/auth.php`

Switch the `api` guard driver from `sanctum` to `jwt`:

```php
'guards' => [
    'web' => [
        'driver'   => 'session',
        'provider' => 'users',
    ],
    'api' => [
        'driver'   => 'jwt',
        'provider' => 'users',
    ],
],
```

### `config/jwt.php`

After publishing, verify the blacklist is enabled:

```php
'blacklist_enabled' => env('JWT_BLACKLIST_ENABLED', true),
'blacklist_grace_period' => env('JWT_BLACKLIST_GRACE_PERIOD', 0),

'providers' => [
    'jwt'     => Tymon\JWTAuth\Providers\JWT\Lcobusc::class,
    'auth'    => Tymon\JWTAuth\Providers\Auth\Illuminate::class,
    'storage' => Tymon\JWTAuth\Providers\Storage\Illuminate::class, // uses Laravel cache
],
```

### `config/hashing.php`

Passwords are hashed using **Argon2id**:

```php
'driver' => env('HASH_DRIVER', 'argon2id'),
```

## User Model

The `User` model must implement `JWTSubject` to work with `tymon/jwt-auth`:

```php
use Tymon\JWTAuth\Contracts\JWTSubject;

class User extends Authenticatable implements JWTSubject
{
    protected $fillable = ['name', 'email', 'password'];

    protected $hidden = ['password', 'remember_token'];

    public function getJWTIdentifier(): mixed
    {
        return $this->getKey();
    }

    public function getJWTCustomClaims(): array
    {
        return [];
    }
}
```

## Repository

### `IUser` Interface

```php
interface IUser
{
    public function getByEmail($email);
    public function store($model);
    public function update($model);
    public function getUserInfo();
}
```

### `UserRepository`

Resolves the authenticated user through the `api` guard (required for JWT):

```php
public function getUserInfo()
{
    return auth('api')->user()->loadCount('posts');
}
```

> **Note:** Always use `auth('api')` instead of `auth()` to target the JWT guard explicitly.

## Service Layer

### `AuthService`

#### `login($request)`

Uses `JWTAuth::attempt()` with raw credentials — no manual password check needed since JWT handles it internally:

```php
$token = JWTAuth::attempt($credentials);
```

Returns:

```php
[
    'token'      => $token,
    'token_type' => 'bearer',
    'expires_in' => auth('api')->factory()->getTTL() * 60, // in seconds
    'user_data'  => new UserResource($user),
]
```

#### `logout($request)`

Invalidates the token server-side via the blacklist (requires `JWT_BLACKLIST_ENABLED=true`):

```php
JWTAuth::invalidate(JWTAuth::getToken());
```

#### `getUserInfo($request)`

Fetches authenticated user with related counts, cached for 1 hour per user:

```php
return Cache::remember('user:info:' . $user->id, now()->addHour(), function () use ($user) {
    return new UserResource($user);
});
```

## Controller

`AuthController` uses `ResponseTrait` for consistent JSON responses. No changes from the Sanctum version are needed in the controller itself — guard switching is handled at the route and service level.

---

## Routes

```php
// routes/api.php
use App\Modules\Authentication\Controllers\AuthController;

Route::prefix('auth')->group(function () {
    Route::post('register', [AuthController::class, 'register']);
    Route::post('login',    [AuthController::class, 'login']);

    Route::middleware('auth:api')->group(function () {
        Route::post('logout',         [AuthController::class, 'logout']);
        Route::post('renew-password', [AuthController::class, 'renewPassword']);
        Route::get('me',              [AuthController::class, 'getUserInfo']);

        // Optional
        Route::post('refresh', [AuthController::class, 'refresh']);
    });
});
```

---

## API Endpoints

| Method | Endpoint | Auth Required | Description |
|--------|----------|:---:|-------------|
| `POST` | `/auth/register` | No | Register a new user |
| `POST` | `/auth/login` | No | Login and receive JWT |
| `POST` | `/auth/logout` | Yes | Invalidate current token |
| `POST` | `/auth/renew-password` | Yes | Update password |
| `GET` | `/auth/me` | Yes | Get authenticated user info |
| `POST` | `/auth/refresh` | Yes | Refresh expired token |

### Request Headers (authenticated routes)

```
Authorization: Bearer <token>
Content-Type: application/json
Accept: application/json
```

## Optional — Token Refresh

Add to `AuthController` to support token refresh before expiry:

```php
public function refresh()
{
    try {
        $newToken = JWTAuth::refresh(JWTAuth::getToken());
        return $this->returnData(__('messages.token.refreshed'), 200, [
            'token'      => $newToken,
            'token_type' => 'bearer',
            'expires_in' => auth('api')->factory()->getTTL() * 60,
        ]);
    } catch (\Exception $e) {
        return $this->returnError($e->getMessage(), 401);
    }
}
```

## JWT vs Sanctum — Key Differences

| | Sanctum | JWT (`tymon/jwt-auth`) |
|---|---|---|
| Token storage | Database (`personal_access_tokens`) | Stateless — no DB row |
| Logout mechanism | Deletes DB row | Blacklists token via cache |
| Guard | `sanctum` | `api` (driver: `jwt`) |
| Resolving user | `auth()->user()` | `auth('api')->user()` |
| Token refresh | Not applicable | `JWTAuth::refresh()` |
| Best for | SPA / same-domain | API / mobile / third-party |

## Error Handling

All exceptions are caught in the controller and returned via `ResponseTrait`:

| Scenario | HTTP Code |
|---|---|
| Email already exists | `422` |
| Invalid credentials | `401` |
| Token creation failure | `500` |
| User not found | `404` |
| Wrong old password | `401` |
| Logout failure | `400` |
