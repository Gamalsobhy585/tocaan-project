# Authentication Module Documentation

## Overview

This module handles all authentication flows using **JWT (JSON Web Tokens)** via the `tymon/jwt-auth` package. It follows a modular architecture with Repository, Service, and Controller layers, using interfaces for dependency injection.

---

## Installation

```bash
composer require tymon/jwt-auth
php artisan vendor:publish --provider="Tymon\JWTAuth\Providers\LaravelServiceProvider"
php artisan jwt:secret
```

---

## Environment Variables

Add the following to your `.env` file:

```env
JWT_SECRET=<auto-generated by php artisan jwt:secret>
JWT_BLACKLIST_ENABLED=true
```

---

## Configuration

### `config/auth.php`

Switch the `api` guard driver from `sanctum` to `jwt`:

```php
'guards' => [
    'web' => [
        'driver'   => 'session',
        'provider' => 'users',
    ],
    'api' => [
        'driver'   => 'jwt',
        'provider' => 'users',
    ],
],
```

### `config/jwt.php`

After publishing, verify the blacklist is enabled:

```php
'blacklist_enabled' => env('JWT_BLACKLIST_ENABLED', true),
'blacklist_grace_period' => env('JWT_BLACKLIST_GRACE_PERIOD', 0),

'providers' => [
    'jwt'     => Tymon\JWTAuth\Providers\JWT\Lcobusc::class,
    'auth'    => Tymon\JWTAuth\Providers\Auth\Illuminate::class,
    'storage' => Tymon\JWTAuth\Providers\Storage\Illuminate::class, // uses Laravel cache
],
```

### `config/hashing.php`

Passwords are hashed using **Argon2id**:

```php
'driver' => env('HASH_DRIVER', 'argon2id'),
```

## Error Handling

All exceptions are caught in the controller and returned via `ResponseTrait`:

| Scenario | HTTP Code |
|---|---|
| Email already exists | `422` |
| Invalid credentials | `401` |
| Token creation failure | `500` |
| User not found | `404` |
| Wrong old password | `401` |
| Logout failure | `400` |
